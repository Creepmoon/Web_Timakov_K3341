Модели данных

Описание структуры базы данных системы управления типографией.

Employee (Сотрудник)

Модель сотрудника типографии, связанная с пользователем системы.

Поля:

  • user (OneToOneField -> User) - связь с пользователем Django
  • position (CharField) - должность:
  • editor - Редактор
  • manager - Менеджер
  • designer - Дизайнер
  • printer - Печатник
  • accountant - Бухгалтер
  • director - Директор
  • phone (CharField) - телефон
  • hire_date (DateField) - дата найма
  • salary (DecimalField) - зарплата
  • department (CharField) - отдел

Author (Автор)

Модель автора книги.

Поля:

  • first_name (CharField) - имя
  • last_name (CharField) - фамилия
  • middle_name (CharField, optional) - отчество
  • birth_date (DateField, optional) - дата рождения
  • email (EmailField, optional) - email
  • phone (CharField, optional) - телефон
  • biography (TextField, optional) - биография
  • created_at (DateTimeField) - дата создания (автоматически)

Связи:

  • books (ManyToMany) - книги автора

Book (Книга)

Модель книги.

Поля:

  • title (CharField) - название
  • isbn (CharField, unique, optional) - ISBN
  • pages (PositiveIntegerField) - количество страниц
  • publication_date (DateField, optional) - дата публикации
  • print_date (DateField, optional) - дата печати
  • print_run (PositiveIntegerField) - тираж
  • price (DecimalField) - цена
  • cost (DecimalField) - себестоимость
  • status (CharField) - статус:
  • draft - Черновик
  • editing - Редактирование
  • design - Дизайн
  • printing - Печать
  • published - Опубликована
  • archived - Архив
  • description (TextField, optional) - описание
  • created_at (DateTimeField) - дата создания (автоматически)
  • updated_at (DateTimeField) - дата обновления (автоматически)

Связи:

  • authors (ManyToMany -> Author) - авторы книги
  • editor (ForeignKey -> Employee) - редактор (только редакторы)
  • financial_records (Reverse -> FinancialStatus) - финансовые записи

Вычисляемые поля:

  • profit - прибыль от книги: (price - cost) * print_run

FinancialStatus (Финансовая запись)

Модель финансовой записи компании.

Поля:

  • date (DateField) - дата операции
  • transaction_type (CharField) - тип операции:
  • income - Доход
  • expense - Расход
  • category (CharField) - категория:
  • book_sales - Продажа книг
  • printing_services - Услуги печати
  • salary - Зарплата
  • materials - Материалы
  • equipment - Оборудование
  • rent - Аренда
  • utilities - Коммунальные услуги
  • other - Прочее
  • amount (DecimalField) - сумма
  • description (TextField, optional) - описание
  • created_at (DateTimeField) - дата создания (автоматически)

Связи:

  • book (ForeignKey -> Book, optional) - связанная книга

Report (Отчёт)

Модель отчёта.

Поля:

  • title (CharField) - название отчёта
  • report_type (CharField) - тип отчёта:
  • financial - Финансовый отчёт
  • books - Отчёт по книгам
  • employees - Отчёт по сотрудникам
  • sales - Отчёт по продажам
  • custom - Пользовательский отчёт
  • start_date (DateField, optional) - начальная дата периода
  • end_date (DateField, optional) - конечная дата периода
  • data (JSONField) - данные отчёта (JSON)
  • created_at (DateTimeField) - дата создания (автоматически)

Связи:

  • created_by (ForeignKey -> Employee) - создатель отчёта

Диаграмма связей

User
  └── Employee (OneToOne)
        ├── Book.editor (ForeignKey)
        └── Report.created_by (ForeignKey)

Author
  └── Book.authors (ManyToMany)

Book
  ├── Author.books (ManyToMany)
  ├── Employee.edited_books (Reverse ForeignKey)
  └── FinancialStatus.book (ForeignKey)

FinancialStatus
  └── Book.financial_records (Reverse ForeignKey)

Report
  └── Employee.reports (Reverse ForeignKey)
