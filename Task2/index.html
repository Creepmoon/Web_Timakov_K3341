В данной лабораторной работе нам необходимо Реализовать веб сервис, в соответствии с вариантом из задания. Вариант задания: 3

Табло отображения информации об авиаперелетах

Хранится информация о номере рейса, авиакомпании, отлете, прилете, типе (прилет, отлет), номере гейта. Необходимо реализовать следующий функционал: - Регистрация новых пользователей. - Просмотр и резервирование мест на рейсах. Пользователь должен иметь возможность редактирования и удаления своих резервирований. - Администратор должен иметь возможность зарегистрировать на рейс пассажира и вписать в систему номер его билета средствами Django-admin. - В клиентской части должна формироваться таблица, отображающая всех пассажиров рейса. - Написание отзывов к рейсам. При добавлении комментариев, должны сохраняться дата рейса, текст комментария, рейтинг (1-10), информация о комментаторе.

Архитектура базы данных

Модель Flight - flight_number - номер рейса (уникальный) - airline - авиакомпания - departure_time - время отлета - arrival_time - время прилета - flight_type - тип рейса (arrival/departure) - gate_number - номер гейта - total_seats - общее количество мест - Метод available_seats() - расчет свободных мест

Модель Reservation (Резервирование)

  • user - пользователь (ForeignKey)
  • flight - рейс (ForeignKey)
  • seat_number - номер места
  • status - статус (pending/confirmed/cancelled)
  • created_at, updated_at - временные метки
  • Уникальность: комбинация flight + seat_number

Модель Ticket (Билет)

  • reservation - резервирование (OneToOne)
  • ticket_number - номер билета (уникальный)
  • passenger_name - имя пассажира
  • passenger_passport - номер паспорта
  • created_at - дата создания

Модель Review (Отзыв)

  • flight - рейс (ForeignKey)
  • user - пользователь (ForeignKey)
  • flight_date - дата рейса
  • text - текст отзыва
  • rating - рейтинг (1-10)
  • created_at - дата создания
  • Уникальность: комбинация flight + user

Функциональность приложения

Аутентификация и авторизация

  • Регистрация пользователей(register_view)
  • Вход выход (Django auth views)
  • защита страниц декоратором @login_required

Просмотр рейсов - Список рейсов(flightListView) - пагинация(10 рейсов на страницу) - Поиск по номеру рейса, авиакомпании, гейту - сортировка по времени отлета

  • Детальная информация о рейсе(flight_detail)
    • Информация о рейсе
    • Список резервирований
    • список отзывов
    • количество свободных мест

Управление резервированиями - Создание резервирования(create_reservation) - Просмотр своих резервирований(my_reservations) - Редактирование резервирования(edit_reservation)

Отзывы о рейсах

  • Создание отзыва(create_review)
  • дата рейса
  • текст отзыва
  • рейтинг(1-10)
  • ограничение один отзыв на рейс от пользователя

Таблица пассажиров - просмотр пассажиров рейса(flight_passengers) - список подтвержденых резевирований - информация о билетах(если имеется)

В данной работе мы используем DjangoORM для работы с данными

URL маршруты

/                          - Список рейсов
/register/                 - Регистрация
/login/                    - Вход
/logout/                   - Выход
/flight/<id>/              - Детали рейса
/flight/<id>/reserve/      - Создание резервирования
/flight/<id>/review/       - Создание отзыва
/flight/<id>/passengers/   - Таблица пассажиров
/reservations/             - Мои резервирования
/reservation/<id>/edit/    - Редактирование резервирования
/reservation/<id>/delete/  - Удаление резервирования
/admin/                    - Административная панель
